上传本地项目到Git/Gitee

  1. 初始化git

git init

  1. 将项目的所有文件添加到缓存中（注意add后有空格）
  1. 将缓存中的文件Commit到git库
  1. 将本地的库链接到远程仓库
  1. 拉取远程仓库代码
  1. 提交代码

可能遇到的问题

  1. fatal: remote origin already exists.解决方法

先输入 git remote rm origin

再输入 git remote add origin 远程仓库地址

  1. git pull 失败，提示：fatal: refusing to merge unrelated histories

方案一 git pull origin master --allow-unrelated-histories

后面加上 –allow-unrelated-histories ， 把两段不相干的 分支进行强行合并 后面再push就可以了

方案二 git push -f -u origin master

强推至远程服务器 上述方式会强制删除远程仓库中冲突代码，即删除创建的README.md文件，不推荐
